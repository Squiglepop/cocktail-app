## Tech Debt Resolved During Retro

### 1. Merge Endpoint Error Pattern (Backend)

**Before:** `merge_ingredients` returned error strings. Router did `isinstance(result, str)` + `result.startswith("Cannot merge")` to determine HTTP status code.

**After:** `merge_ingredients` raises `LookupError` (not found ‚Üí 404) and `ValueError` (validation ‚Üí 400). Router catches exceptions. Return type simplified to `Tuple[int, int]`.

**Files changed:**
- `backend/app/services/ingredient_service.py` ‚Äî Raise exceptions instead of returning strings
- `backend/app/routers/admin.py` ‚Äî Catch `ValueError`/`LookupError` instead of string matching

**Verification:** 496 backend tests passing, 0 regressions.

### 2. Documentation Updates to `project_context.md`

Three sections added/updated:
- **Adversarial Code Review Guidance** ‚Äî Calibrates expectations for multi-round reviews (rounds 1-2 catch bugs, round 3+ finds polish)
- **SQLAlchemy `flush()` before delete pattern** ‚Äî Mandatory pattern with code example
- **Service‚ÜíRouter error convention** ‚Äî `ValueError`/`LookupError`/`None` pattern documented with examples
- **Python 3.9 type hint constraints** ‚Äî `Union[X, Y]` not `X | Y`
- **Don't-Miss Rules** ‚Äî 2 new rows (error strings, flush before delete)

### 3. Frontend `act()` Warnings

**Before:** 4+ `act()` warnings in `recipe-detail.test.tsx` (FavouritesProvider, RecipeDetailPage) and `Header.test.tsx` (AuthProvider).

**After:** Zero warnings. Fixed by wrapping async state-triggering clicks in `act()` and flushing async provider effects. Also fixed broken assertion (`/served up/i` ‚Üí `/\bUp\b/` to match `formatEnumValue('up')` output).

**Files changed:**
- `frontend/tests/pages/recipe-detail.test.tsx` ‚Äî 18 tests, 0 warnings
- `frontend/tests/components/Header.test.tsx` ‚Äî 15 tests, 0 warnings
