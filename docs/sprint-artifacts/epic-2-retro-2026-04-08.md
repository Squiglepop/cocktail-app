## Challenges and Growth Areas

1. **Code review still the primary quality gate (3/3 stories)** — Pre-review checklist reduced severity but didn't eliminate gaps. Story 2-1 needed 3 review rounds.

2. **Adversarial review diminishing returns** — Story 2-1 Review 3 found polish/refactoring items (null validator, method signature refactoring) not crash bugs. The adversarial mandate to always find 3-10 problems inflates severity labels on later rounds. Team should evaluate findings by actual impact.

3. **Error-passing convention inconsistency** — Story 2-1 service returns `None` on failure (caller raises HTTP error). Story 2-3 service returns error strings (caller does string matching for status code). Two different patterns in the same service file.

4. **Python 3.9 compatibility gotcha** — `Tuple[int, int] | str` syntax requires 3.10+. Had to use `Union[Tuple[int, int], str]`.

5. **SQLAlchemy ORM flush behavior** — `db.delete(source)` triggers FK nullification on related rows even after those rows were already updated/deleted. Requires `db.flush()` before deleting parent records.

6. **Test fixture workarounds** — Story 2-2 duplicate detection tests couldn't use the create API (rejects case-insensitive duplicates with 409). Required direct DB insertion via helper function.

---

## Key Insights

1. **Adversarial code review has diminishing returns per round but cumulative value** — Keep the process. Round 1-2 catches real bugs. Round 3+ catches polish. Evaluate by actual impact, not severity label.

2. **Epic 1 retro action items measurably shifted quality left** — Fewer crash bugs per story compared to Epic 1. Checklist + defensive patterns in specs = fewer surprises.

3. **Small, focused epics deliver faster and cleaner** — 3 stories vs 6 stories. Clear pipeline. Less context switching. Tighter feedback loops.

4. **Existing architecture solved the hard problem for free** — `get_current_user` already checks `is_active` on every request. Token invalidation for Epic 3's user deactivation is already handled. No new infrastructure needed.
